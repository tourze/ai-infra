# Git 日期版本差异分析

## 语言要求

**重要：** 在整个执行过程中，必须使用中文进行所有交互、输出和说明。

## 概述

此技能基于日期分析 Git 代码差异，包括：
- 解析输入日期，查找该日期0点之后最近的一次提交
- 对比该提交与当前代码 (HEAD)
- 列出所有变更文件及变更类型
- 分析代码逻辑变更
- 检测数据库结构变更（迁移、表结构变化）
- 生成完整的差异说明报告

## 使用场景

当用户提到以下需求时使用：
- "看看今天改了什么代码"
- "对比2024年1月15日到现在有什么改动"
- "检查昨天的代码变更"
- "查看指定日期之后的变化"
- "生成日报/变更报告"

## 工作流程

### 步骤 1：解析输入日期

**输入格式支持：**
- 绝对日期：`2024-01-15`、`2024-01-15 00:00:00`
- 相对日期：`yesterday`（昨天）、`today`（今天）
- 中文日期：`2024年1月15日`

**解析逻辑：**
1. 解析用户输入的日期
2. 将日期设置为该天的 00:00:00
3. 向用户确认解析结果

**输出格式：**
```
⏺ 正在解析日期...

📅 目标日期：2024-01-15 00:00:00（星期一）
```

### 步骤 2：查找目标提交

**执行命令查找该日期之后最近的提交：**

```bash
# 获取该日期0点之后的第一条提交记录
git log --since="2024-01-15 00:00:00" --format="%H|%ai|%s" --reverse | head -1
```

**解析输出：**
- 提取 commit hash
- 提取提交时间
- 提取提交信息

**如果找不到提交：**
```bash
# 尝试向前查找最近的提交
git log --until="2024-01-15 00:00:00" --format="%H|%ai|%s" -1
```

**输出格式：**
```
🔍 找到目标提交：
  Commit: a1b2c3d4e5f6...
  时间：2024-01-15 09:23:45
  信息：修复用户登录问题
```

### 步骤 3：获取变更文件列表

**执行命令获取文件变更：**

```bash
# 获取变更文件及状态
git diff --name-status <TARGET_COMMIT> HEAD

# 获取统计信息
git diff --stat <TARGET_COMMIT> HEAD
```

**变更状态说明：**
- `M` = 修改（Modified）
- `A` = 新增（Added）
- `D` = 删除（Deleted）
- `R100` = 重命名（Renamed）

**输出格式：**
```
📁 变更文件统计：
  - 总计：N 个文件
  - 新增：N 个
  - 修改：N 个
  - 删除：N 个
  - 重命名：N 个
```

### 步骤 4：分类变更文件

**按文件类型分类：**

**后端代码：**
- `.php`、`.py`、`.go`、`.java`、`.js`、`.ts`
- API 路由、控制器、服务层

**前端代码：**
- `.vue`、`.tsx`、`.jsx`
- 组件、视图

**数据库相关：**
- `.sql` - SQL 脚本
- `migrations/` - 迁移目录
- `*/schema.sql` - 结构文件
- `Entity.php`、`Model.php` - 模型文件

**配置文件：**
- `.env`、`.yaml`、`.json`、`.toml`

**测试文件：**
- `.test.php`、`.spec.ts`、`__tests__/`

**其他：**
- `.md` - 文档
- `.css`、`.scss` - 样式

### 步骤 5：检查数据库变更

**重点检查以下文件类型：**

1. **SQL 迁移文件：**
   ```bash
   git diff <TARGET_COMMIT> HEAD -- "*.sql" "migrations/*"
   ```

2. **实体/模型文件：**
   ```bash
   git diff <TARGET_COMMIT> HEAD -- "*/Entity/*.php" "*/Model/*.php"
   ```

3. **数据库配置：**
   ```bash
   git diff <TARGET_COMMIT> HEAD -- "config/database.php"
   ```

**分析数据库变更类型：**
- 新建表（CREATE TABLE）
- 修改表结构（ALTER TABLE）
- 添加字段
- 修改字段类型
- 添加索引
- 删除表/字段

**输出格式：**
```

【新建表】

【修改表结构】

```

## 实现细节

### 日期解析逻辑

**支持的日期格式：**

```python
# 绝对日期
"2024-01-15"
"2024-01-15 00:00:00"
"2024年1月15日"

# 相对日期
"today" -> 今天 00:00:00
"yesterday" -> 昨天 00:00:00
"2 days ago" -> 2天前 00:00:00
```

### 查找目标提交的算法

```bash
# 步骤1：尝试查找该日期0点之后的提交
git log --since="$DATE 00:00:00" --format="%H|%ai|%s" --reverse | head -1

# 步骤2：如果没有找到，查找该日期之前的最近提交
git log --until="$DATE 00:00:00" --format="%H|%ai|%s" -1

# 步骤3：验证提交是否有效
git cat-file -e $COMMIT^{commit}
```

### 数据库变更检测

**检测规则：**

1. **SQL 文件变更：**
   - 文件路径匹配：`*.sql`、`migrations/*.php`
   - 内容包含：`CREATE TABLE`、`ALTER TABLE`、`DROP TABLE`

2. **模型/实体文件变更：**
   - 文件路径匹配：`*/Entity/*.php`、`*/Model/*.php`
   - 分析属性/字段定义的变化

3. **配置文件变更：**
   - 数据库连接配置
   - ORM 配置

**风险评估：**

| 变更类型 | 风险级别 | 说明 |
|---------|---------|------|
| 删除表 | 高 | 可能导致数据丢失 |
| 修改字段类型 | 高 | 可能导致数据截断或转换错误 |
| 添加必填字段 | 中 | 需要为现有数据提供默认值 |
| 新建表 | 低 | 不影响现有数据 |
| 添加索引 | 低 | 可能影响写入性能 |

### 逻辑差异分析要点

**关注重点：**

1. **核心业务逻辑：**
   - 支付、订单、用户相关
   - 权限、认证逻辑

2. **API 接口：**
   - 参数变更（破坏性）
   - 返回值变更
   - 新增接口

3. **性能影响：**
   - 数据库查询优化
   - 缓存策略
   - 异步处理

4. **安全性：**
   - 输入验证
   - 权限检查
   - 敏感数据处理

## 错误处理

### 无效日期

```
❌ 错误：无法解析日期 "xxx"

支持的日期格式：
  • 2024-01-15
  • 2024-01-15 00:00:00
  • 2024年1月15日
  • today, yesterday
```

### 找不到提交

```
❌ 错误：在指定日期之后没有找到提交

可能的原因：
  • 该日期之后没有代码提交
  • Git 历史记录不完整

建议：
  • 使用 git log 查看提交历史
  • 尝试使用更早的日期
```

### 无变更

```
ℹ️ 提示：指定日期与当前版本之间没有差异

可能的情况：
  • 代码没有变更
  • 变更已提交但未合并
```

## 配置选项

### 自定义日期格式

在项目中可以配置首选日期格式：

```yaml
# .git-version-diff.yml
date_format: "Y-m-d H:i:s"
timezone: "Asia/Shanghai"
```

### 忽略文件

配置需要忽略的文件或目录：

```yaml
# .git-version-diff.yml
ignore:
  - "vendor/"
  - "node_modules/"
  - "*.log"
  - ".gitignore"
```

### 关注路径

配置需要重点关注的路径：

```yaml
# .git-version-diff.yml
focus:
  - "src/"
  - "app/"
  - "database/"
```
